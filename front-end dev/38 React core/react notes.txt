2. React Using CDN (No Build Tools)
Basic Setup
<!DOCTYPE html>
<html>
<head>
  <title>React CDN Example</title>
</head>
<body>

  <div id="root"></div>

  <!-- React -->
  <script src="https://unpkg.com/react@18/umd/react.development.js"></script>

  <!-- React DOM -->
  <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>

  <!-- Babel -->
  <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

  <script type="text/babel">
    function App() {
      return <h1>Hello React</h1>;
    }

    const root = ReactDOM.createRoot(document.getElementById("root"));
    root.render(<App />);
  </script>

</body>
</html>
Important Points

React CDN setup includes:

React

ReactDOM

Babel

✔ Use for: Learning, demos
❌ Not for production

3. Virtual DOM
Normal DOM Flow
User Action → JS updates DOM → Browser re-render

❌ Slow & expensive

Virtual DOM Flow
State change
   ↓
New Virtual DOM
   ↓
Compare with old Virtual DOM
   ↓
Find differences
   ↓
Update only changed parts
Advantages

Faster updates

Less DOM manipulation

Better performance

4. React vs ReactDOM
React (Core Library)

Package: react

Works:

Create components

Manage state

Virtual DOM

Hooks

import React from "react";
ReactDOM

Package: react-dom

Works:

Connect React with browser DOM

Render UI

import ReactDOM from "react-dom";
ReactDOM.createRoot(document.getElementById("root")).render(<App />);
5. React.createElement()
Syntax
React.createElement(type, props, children)
Example
const element = React.createElement(
  "h1",
  null,
  "Hello React"
);
Render
ReactDOM.render(element, document.getElementById("root"));
JSX Behind the Scenes

JSX:

<h1>Hello</h1>

Converted to:

React.createElement("h1", null, "Hello");

✔ JSX = syntactic sugar

6. Babel

Babel = JavaScript Compiler

Example

JSX:

const element = <h1>Hello</h1>;

Converted to:

const element = React.createElement("h1", null, "Hello");
Why Babel?

Browsers don’t understand JSX

JSX → JavaScript
ES6+ → ES5
7. Reconciliation

React’s process to update UI efficiently.

Flow
State change
   ↓
New Virtual DOM
   ↓
Compare with old
   ↓
Find differences
   ↓
Update real DOM
Example

Old:

<h1>Hello</h1>

New:

<h1>Hello Mizan</h1>

✔ Only text updated (not whole element)

8. Diffing Algorithm

React compares two Virtual DOM trees

✔ Complexity: O(n) (fast)

React Assumptions
1. Different elements → new tree
<div> → <span>

✔ Old removed, new created

2. Keys for lists
{items.map(item => (
  <li key={item.id}>{item.name}</li>
))}

✔ Helps React identify:

Added items

Removed items

Updated items
