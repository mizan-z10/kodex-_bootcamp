

# 1. Absolute Units (Not good for responsive design)

These units have **fixed size**. They do not change based on screen.

| Unit | Meaning     | Example            |
| ---- | ----------- | ------------------ |
| `px` | Pixels      | `font-size: 16px;` |
| `cm` | Centimeters | `width: 10cm;`     |
| `mm` | Millimeters | `width: 5mm;`      |
| `in` | Inches      | `width: 1in;`      |
| `pt` | Points      | `font-size: 12pt;` |

Example:

```css
.box{
width:300px;
height:200px;
}
```

Problem:
On mobile **300px may be too big or too small**.

So professionals **avoid heavy use of px for layouts**.

---

# 2. Relative Units (Best for responsive design)

These units **change based on screen or parent element**.

## 1. Percentage `%`

Depends on **parent element size**.

```css
.container{
width:100%;
}

.box{
width:50%;
}
```

Meaning:

* container = full width
* box = half of container

---

## 2. `em`

Depends on **parent font size**.

Example:

```css
.parent{
font-size:20px;
}

.child{
font-size:2em;
}
```

Result:

```
2 × 20px = 40px
```

Problem:
It keeps multiplying in nested elements.

---

## 3. `rem` (Better than em)

Depends on **root element (html)**.

```css
html{
font-size:16px;
}

h1{
font-size:2rem;
}
```

Result:

```
2 × 16px = 32px
```

This is **most used for fonts**.

---

# 3. Viewport Units (Very important for responsive)

Viewport = **screen size**

| Unit   | Meaning             |
| ------ | ------------------- |
| `vw`   | % of screen width   |
| `vh`   | % of screen height  |
| `vmin` | smaller of vw or vh |
| `vmax` | larger of vw or vh  |

Example:

```css
.hero{
height:100vh;
}
```

Meaning:

```
Hero section = full screen height
```

Example:

```css
h1{
font-size:5vw;
}
```

Font grows when screen grows.

---

# 4. Best Units Professionals Use

For real websites developers mostly use:

```
px → borders, small spacing
rem → font sizes
% → layout width
vw/vh → full screen sections
```

Example professional layout:

```css
.container{
width:90%;
max-width:1200px;
margin:auto;
}

h1{
font-size:2.5rem;
}

.section{
height:100vh;
}
```

---

# 5. Real Responsive Example

```css
.container{
width:90%;
margin:auto;
}

.card{
width:100%;
max-width:400px;
padding:2rem;
}

h1{
font-size:3vw;
}
```

This layout will adapt to **mobile, tablet, desktop automatically**.

---

# Brutal Truth (Important)

Right now many beginners (maybe you too) do this mistake:

```
width:300px
height:400px
margin-left:200px
```

That **destroys responsiveness**.

Instead think like this:

```
width:100%
max-width
flex
grid
rem
vw
```

| Unit   | Type     | Meaning                            | Example           | Use                       |
| ------ | -------- | ---------------------------------- | ----------------- | ------------------------- |
| `px`   | Absolute | Fixed pixels                       | `width:300px;`    | Borders, small spacing    |
| `%`    | Relative | Depends on parent size             | `width:50%;`      | Layout width              |
| `em`   | Relative | Depends on parent font size        | `font-size:2em;`  | Nested elements           |
| `rem`  | Relative | Depends on root (`html`) font size | `font-size:2rem;` | Fonts (most common)       |
| `vw`   | Viewport | % of screen width                  | `width:50vw;`     | Responsive width          |
| `vh`   | Viewport | % of screen height                 | `height:100vh;`   | Full screen sections      |
| `vmin` | Viewport | Smaller of `vw` or `vh`            | `width:50vmin;`   | Responsive scaling        |
| `vmax` | Viewport | Larger of `vw` or `vh`             | `width:50vmax;`   | Large responsive elements |
